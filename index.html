Finding longest selector
(function(j, w) {
  w.call(document.styleSheets).forEach(function(q) {
    try { /* Accessing remote domain cssRules fails. */
      w.call(q.rules || q.cssRules).forEach(function(k) {
        (k.selectorText || '').split(',').forEach(function(b) {
          (b.length > j.length) && (j = b);
        });
      });
    } catch (z) {}
  });
  return j;
})('', [].slice);

BEM - Block Element Modifier
  • Methodology for naming and structuring selectors.
  • Uses classes, avoids tagnames, ids
  • Harry Roberts' style
Block
  • Logically and functionally independent page component.
  • A block encapsulates behaviour (JavaScript, HTML, CSS).
  • Blocks allow reuse.

.block{
  background: #def;
}
.block{
  background: #def;
}
Element
  • Inseparable part of a block.

.block{
  background: #def;

  >.element{
    box-shadow: 2px;
}
.block{
  background: #def;
}
.block__element{
  box-shadow: 2px;
}
Modifier
  • BEM entity that defines the appearance and behavior of a block or an element.
  • Modifiers are similar in essence to HTML attributes.

.block{
  background: #def;

  >.element{
    box-shadow: 2px;

    &.is-selectable{
      cursor: pointer;
      &:hover{
        background: #0f0;
      }
    }
  }
}
.block{
  background: #def;
}
.block__element{
  box-shadow: 2px;
}
.block__element--selectable{
  @extend .block__element;
  cursor: pointer;
  &:hover{
    background: #0f0;
  }
}
SMACSS
  • Scalable and Modular Architecture for CSS
  • Categorizes rules into 5 types, aka. namespaces
  • Promotes similar patterns for selectors
Base rules
  • The defaults
  • Normalize, reset
  • Always apply to selected element on the page
  • Never requires !important

body{
  margin: 0;
}
a{
  outline: none;
}
Layout rules
  • Divide the page into sections
  • Hold one or more modules together

.l-header{
  positon: fixed;
  top:0;
  left:0;
  right:0;
}
Module rules
  • Reusable, modular parts of our design
  • They are the callouts, the sidebar sections, the product lists and so on

.comments{
  position: sticky;
}
.m-comments{
  position: sticky;
}
State rules
  • Describe how a module or layout will look when in a particular state (hidden, expanded)
  • Describe how a module or layout looks on screens that are smaller or bigger
  • They are also about describing how a module might look in different views like the home page or the inside page.
  • Always use !important to force the state.

.is-hidden{
  display: none !important;
}

.is-text-right{
  text-align: right !important;
}
Theme rules
  • Similar to state rules in that they describe how modules or layouts might look
  • Most sites don’t require a layer of theming but it is good to be aware of it.

.t-border{
  border-color: $brandColor;
}
.t-text{
  color: $brandColor;
}
.t-background{
  background-color: $brandColor;
}

  • Custom namespaces
  • State classes
  • No more than 3 nesting

Debugging
@mixin debug($color){
  outline: 4px solid $color !important;
}

/* Focus, Shame */
:focus, [class^="__"],[class*=" __"] { @include debug(#f00); }

/* Layout */
[class^="l-"], [class*=" l-"] { @include debug(#0f0); }

/* Component */
[class^="c-"],[class*=" c-"] { @include debug(#00f); }

/* State/utils */
[class^="is-"], [class*=" is-"],
[class^="has-"], [class*=" has-"] { @include debug(#ff0); }

/* Slide */
[class^="s-"], [class*=" s-"] { @include debug(#f0f); }

/* Javascript */
[class^="js-"],[class*=" js-"] { @include debug(#0ff); }
